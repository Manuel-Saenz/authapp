Auth Service

User Manual — Email + Password + Google Authenticator (TOTP)

Overview

The Auth Service verifies a user's identity using three factors:

1Email address — verified during registration
2Password — a secret only you know
3One-time code — 6 digits from Google Authenticator

All three must be correct for authentication to succeed. The service offers two interfaces:

Browser UI

Forms and QR code rendered in your browser. No tools required.

http://localhost:8000/

JSON API

Programmatic access and Swagger docs for developers.

http://localhost:8000/docs

Getting Started

What you need

Google Authenticator installed on your phone:

  • Android — Google Play
  • iPhone — App Store
Any TOTP-compatible app works — Authy, Microsoft Authenticator, 1Password, etc.

Start the server

cd auth_app
python -m uvicorn main:app --reload

Then open http://localhost:8000/ in your browser.

Browser UI Guide

1

Register

Open http://localhost:8000/ and fill in the registration form:

Enter your email address, then choose a password. Click the eye icon on the right of the password field to show or hide what you are typing. Enter the same password again in the Confirm password field — the form will alert you if they do not match before submitting.

Click Register. The service validates your inputs and sends a verification email to the address you provided.

2

Verify your email

Check your inbox for an email with the subject "Verify your email — AuthService" and click the link inside. The link is valid for 30 minutes.

If it expires, go back to the registration page and register again to get a new link.

3

Scan the QR code

After clicking the verification link, your browser shows a QR code. Open Google Authenticator on your phone. Tap + → Scan a QR code and point your camera at the QR code on screen. The app will immediately start showing a 6-digit code that refreshes every 30 seconds.

Can't scan? Tap + → Enter a setup key instead. The QR code encodes a URI like:
otpauth://totp/AuthService:you%40example.com?secret=YOURSECRET&issuer=AuthService
Enter your email as the account name and the secret value as the key (type: Time-based).

4

Confirm setup

Click I've scanned it on the QR page. Enter the 6-digit code currently shown in Google Authenticator and click Confirm. A success message confirms your account is ready.

5

Log in

Click Go to login or open http://localhost:8000/ui/login. Enter your email, your password (use the eye icon to check what you typed), and the current 6-digit code from Google Authenticator, then click Log in.

API Guide

Use these endpoints via http://localhost:8000/docs (Swagger UI) or with curl.

The JSON API does not require email verification — it returns the TOTP URI directly. Email verification applies only to the browser UI flow.

Register

curl -X POST http://localhost:8000/register \
  -H "Content-Type: application/json" \
  -d '{"email": "you@example.com", "password": "your-password"}'

Response (201):

{
  "status": "ok",
  "totp_uri": "otpauth://totp/AuthService:you%40example.com?secret=XXXX&issuer=AuthService",
  "message": "Scan this URI with Google Authenticator, then call /setup-confirm"
}

To generate the QR code locally from the totp_uri:

python -c "import qrcode; qrcode.make('PASTE_URI_HERE').save('qr.png')"

Confirm TOTP setup

curl -X POST http://localhost:8000/setup-confirm \
  -H "Content-Type: application/json" \
  -d '{"email": "you@example.com", "totp_code": "123456"}'

Response (200): {"status": "ok"}

Authenticate

curl -X POST http://localhost:8000/authenticate \
  -H "Content-Type: application/json" \
  -d '{"email": "you@example.com", "password": "your-password", "totp_code": "123456"}'

Success: {"status": "ok"}

Failure: {"status": "error", "message": "Invalid credentials or OTP code"}

Email Configuration

Email verification is sent via Resend (recommended for production) or SMTP (local dev fallback).

Resend — production (Railway)

Resend uses an HTTP API that works on all cloud platforms. Set these two variables in Railway's Variables panel:

VariableValueDescription
RESEND_API_KEYre_xxxxxxxxxxxxAPI key from resend.com dashboard
SMTP_FROMnoreply@yourdomain.comSender address — must be on a domain verified in Resend

Resend domain setup (one-time)

  1. Sign up at resend.com and go to Domains → Add Domain
  2. Add the DNS records Resend provides — three records across two subdomains (resend._domainkey and send)
  3. Click Verify in Resend once the records have propagated (5–30 minutes)
  4. Go to API Keys → Create API Key (Full access) and copy the key
  5. Set RESEND_API_KEY in Railway and redeploy
Local development: If neither RESEND_API_KEY nor SMTP_HOST is set, the verification link is shown directly on the page after registration. No email is sent. This is intentional for local testing.

Security Rules

Email must be verified

Account creation in the browser UI is not complete until the user clicks the verification link sent to their email address.

Verification links are single-use and expire

Each link works exactly once and expires after 30 minutes. If the link expires, the user must register again.

Codes are single-use

Each 6-digit code works only once, even within the same 30-second window. A second attempt with the same code will fail.

Codes expire every 30 seconds

Always use the code currently shown in Google Authenticator. A small clock icon in the app shows how many seconds remain.

Clock skew tolerance

Codes from the window immediately before or after the current one are accepted, to allow for slight phone clock drift. If codes are always rejected, check your phone's time is set to automatic.

Brute-force lock

5 consecutive failures lock the account for 15 minutes. A successful login resets the counter.

Timing protection

The service takes the same time to respond whether or not an email exists, preventing user enumeration by measuring response times.

Troubleshooting

"Passwords do not match" alert on the registration form

Both password fields must contain exactly the same text. Use the eye icon on either field to reveal what you typed and check for typos.

Verification email never arrives

  • Check your spam/junk folder
  • The link expires in 30 minutes — if you waited too long, register again
  • Ask the administrator to confirm that SMTP is correctly configured

"This verification link has expired or has already been used"

Links are valid for 30 minutes and can only be clicked once. Go back to the registration page and register again to get a new link.

"Invalid credentials or OTP code" even though everything looks right

  • Use the code currently shown in Google Authenticator — codes expire every 30 seconds
  • You may have already used that code — wait for the next one
  • Check your phone's date and time are set to automatic / network time
  • Check for extra spaces in your email or password — use the eye icon to verify the password field

"TOTP setup not completed"

Complete the confirm step at http://localhost:8000/ui/confirm using a valid code from Google Authenticator.

Lost access to Google Authenticator

An administrator can reset your credentials from the admin panel. After a reset, you can register again with the same email address to set up a new TOTP entry.

Google Authenticator shows a code but it is always rejected

Your phone clock has drifted.

  • Android: Settings → General Management → Date and Time → enable Automatic date and time
  • iPhone: Settings → General → Date & Time → enable Set Automatically

The service is not running

cd auth_app
python -m uvicorn main:app --reload
